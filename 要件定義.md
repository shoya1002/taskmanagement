# Trello風タスク管理アプリ 要件定義書

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| アプリ名 | Trello風タスク管理アプリ |
| 作成日 | 2026-05-18 |
| 目的 | タスクをカード形式で管理し、進捗を視覚的に把握する |
| 対象ユーザー | 個人（1人利用） |

---

## 2. 目的・背景

タスクをTrelloのようにカード形式で管理し、「Todo / 進行中 / 完了」の3列で進捗を視覚的に把握できる個人用タスク管理アプリを作成する。

---

## 3. 機能要件

### 3-1. 必須機能（初回リリース対象）

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | リスト表示 | Todo / 進行中 / 完了 の3列を画面に表示する |
| 2 | カード追加 | 各列にテキスト入力でタスク（カード）を追加できる |
| 3 | カード削除 | カードの削除ボタン（×）を押すと該当カードを削除できる |
| 4 | カード移動 | カードに付いたボタンで別の列へ移動できる |
| 5 | データ保存 | ページを閉じてもデータが消えないよう、ブラウザに自動保存する |

### 3-2. 将来対応機能（初回リリース対象外）

| # | 機能名 | 説明 |
|---|--------|------|
| 1 | カード編集 | 追加済みカードのテキストを後から変更できる |
| 2 | 期限設定 | カードに締め切り日を設定・表示できる |
| 3 | ドラッグ＆ドロップ移動 | カードをドラッグして別の列に移動できる |

---

## 4. 非機能要件

| 項目 | 内容 |
|------|------|
| 対応環境 | モダンブラウザ（Chrome / Firefox / Safari 最新版） |
| データ共有 | 不要（個人利用のみ） |
| サーバー | 不要（ブラウザのみで動作） |
| レスポンシブ | 対応しない（デスクトップのみ） |

---

---

## 6. 技術選定

| 項目 | 選択 | 理由 |
|------|------|------|
| 言語 | JavaScript | Web標準言語 |
| フレームワーク | React | UIを部品単位で管理しやすい |
| データ保存 | localStorage | サーバー不要、ブラウザに自動保存 |
| スタイル | CSS | シンプルに見た目を整える |

---

## 7. データ構造

アプリが扱うデータの形（localStorageに保存する形式）。

```json
{
  "columns": [
    {
      "id": "todo",
      "title": "Todo",
      "cards": [
        { "id": "card-1", "text": "タスクA" },
        { "id": "card-2", "text": "タスクB" }
      ]
    },
    {
      "id": "in-progress",
      "title": "進行中",
      "cards": [
        { "id": "card-3", "text": "タスクC" }
      ]
    },
    {
      "id": "done",
      "title": "完了",
      "cards": [
        { "id": "card-4", "text": "タスクD" }
      ]
    }
  ]
}
```
